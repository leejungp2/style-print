사용 모델과 API 구성도

OpenAI Responses API

  • URL: /v1/responses
  • 기본 모델: gpt-4.1-mini
  • facet 분석: 이미지 입력 detail: high + JSON schema
  • 코드 감사: 생성 코드 + IntentSpec 비교
  • 일관성 judge: OPENAI_JUDGE_MODEL fallback

v0 API

  • URL: https://api.v0.dev/v1/chats
  • 모델: v0-mini
  • 모드: responseMode: sync
  • 옵션: 이미지 생성 off · thinking off
  • 출력: TSX/JS/CSS 파일 · 선택된 UI entry file

Fastify API 경로

  • /api/references/upload · 이미지 저장
  • /api/facets/extract · FacetPack
  • /api/recipes/recommend · recipe 순위화
  • /api/intents/evaluate · 점수 + judge
  • /api/generate/v0 · 생성 작업
  • /api/audit/analyze · FacetDiff

로컬 도구

  • sharp · 픽셀 기반 색상 추출
  • esbuild · 미리보기 번들
  • Playwright · 미리보기 스크린샷
  • Vitest · API/컴포넌트 테스트

레퍼런스 분석 원리: 규칙 기반 + LLM

규칙 기반

결정적 추출

  • 이미지 버퍼 → base64 data URL
  • sharp 색상 샘플링 · 상위 6개 색상
  • 의미 역할 부여: primary · secondary · background · surface · text
  • 대비 비율 계산 · WCAG 기준 확인
  • 업로드 메타데이터 · width/height context
LLM 비전

구조화된 facet 분석

  • 입력: 스크린샷 이미지 + 추출 palette
  • 스키마 출력: typography · layout · spacing · componentStyle
  • mood keywords: 소스 조화도 계산 입력
  • strict JSON schema · 자유 형식 parsing 없음
  • confidence/evidence → 출처 배지

일관성 점수 로직: 규칙 기반 기준

일관성 점수 = clamp(100 - Σ dimension 감점)
접근성text/background < 4.5:1 · primary/background < 3:1
시각 일관성compact + body < 14px · 간격/밀도 불일치
의도 커버리지palette · typography · layout · spacing 누락
출처 커버리지선택 source 근거 누락
소스 조화도mood keyword Dice similarity · 낮은 confidence source
생성 준비도빈 프롬프트 · 빈 화면 계획
수정 계획
  • 대비 수정: 텍스트 색상을 WCAG AA 기준으로 조정
  • 타이포그래피 수정: compact layout body 최소 14px
  • 간격 수정: compact 4px · comfortable 8px
Recipe 순위화
  • 후보 조합 ≤ 200
  • 정렬: 일관성 점수 높은 순
  • tie-break: 섞인 source 수가 적은 순

LLM Judge 로직: 보조 평가

1
기준 평가
규칙 finding · dimension score
2
프롬프트
rubric · checklist · calibration anchor
3
OpenAI Judge
ordinal rating · strict JSON
4
정규화
rating + checklist → 점수
5
저장
CoherenceJudgeResult
6
피드백
정확함 · 너무 높음 · 너무 낮음

Judge 프롬프트 가드레일

  • version: coherence-judge-v2
  • rating: strong · adequate · weak · fail
  • 규칙 finding = 사실 힌트, 숫자 anchor 아님
  • 출처 근거 임의 생성 금지

운영 모드

  • judgeMode: shadow frontend 평가 호출에 포함
  • primary mode disabled
  • judge 실패 → warning log · baseline result 유지
  • feedback 저장: coherence-feedback.json
