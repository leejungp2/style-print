시스템 아키텍처: 5개 레이어와 책임

01

Frontend

Vite React + TypeScript. 사용자는 업로드, 레시피 선택, 생성 결과, audit 결과를 확인합니다.

02

Backend API

Fastify API가 upload, extract, intent, evaluate, generate, audit 요청을 받습니다.

03

Agent Orchestrator

facet 추출, recipe 추천, coherence 평가, repair plan, generation job을 순서대로 연결합니다.

04

Tools & Models

sharp, OpenAI, v0, esbuild, Playwright가 실제 분석, 생성, 미리보기, 감사를 수행합니다.

05

Runtime Artifacts

ReferenceAsset, FacetPack, IntentSpec, GeneratedCode, AuditReport가 단계별 기록으로 남습니다.

앞 슬라이드의 6단계 파이프라인은 이 5개 레이어를 통과하며 실행되고, 산출물은 현재 data/*.json과 public/uploads에 저장됩니다.
레이어별 핵심 계약
  • Frontend는 사용자가 업로드, 레시피 선택, 생성 결과, audit 결과를 확인하는 표면입니다.
  • Backend API는 /api/references/upload, /api/facets/extract, /api/intents/create, /api/intents/evaluate, /api/generate/v0, /api/audit/analyze를 제공합니다.
  • Agent Orchestrator는 분석, 추천, 검증, repair, 생성 job을 단계별로 연결합니다.
  • Tools and Models는 sharp, OpenAI Responses API, v0 API, Playwright로 실제 처리를 수행합니다.
  • Runtime Artifacts는 각 단계의 결과를 저장해 다음 단계 입력과 provenance로 사용합니다.

사용자 시나리오: 화면 조작과 내부 처리

사용자 시나리오 한 줄 흐름

화면에서는 단순한 업로드처럼 보이지만, 내부에서는 분석, 추천, 검증, 생성, 감사가 단계적으로 이어집니다.
01
사용자 입력

레퍼런스 이미지를 올리고 추천 레시피를 고릅니다.

02
웹 요청

/api/references/upload → /api/facets/extract → /api/intents/create 순서로 상태를 만듭니다.

03
패싯 정리

ReferenceAsset와 FacetPack을 만들고 디자인 특징을 구조화합니다.

04
의도 정규화

/api/intents/evaluate가 IntentSpec의 coherence와 repair plan을 점검합니다.

05
코드 생성

/api/generate/v0가 generation job을 만들고 v0로 React + Tailwind UI를 생성합니다.

06
감사 확인

/api/audit/analyze가 GeneratedCode와 AuditReport를 연결합니다.

사용자 관점

  • 레퍼런스 업로드
  • 추천 레시피 선택 또는 패싯 출처 직접 조정
  • 미리보기, 코드, 감사를 순서대로 확인

내부 처리

  • sharp는 이미지 픽셀 기반 색상을 추출
  • OpenAI는 facet 분석, coherence judge, audit을 담당
  • v0는 실제 React + Tailwind 코드를 생성
  • 중간 산출물은 ReferenceAsset → FacetPack → IntentSpec → GeneratedCode → AuditReport로 연결

핵심 메시지

  • 화면은 단순
  • 내부는 분석 → 추천 → 검증/repair → 생성 → 감사
  • 결과보다 과정이 추적 가능하도록 설계
한 장으로 보면, 사용자는 업로드와 확인만 하지만 시스템은 API와 tool을 거치며 단계별로 상태를 축적합니다.

규칙 기반

결정적 추출

  • 이미지 버퍼 → base64 data URL
  • sharp 색상 샘플링 · 상위 6개 색상

  • 대비 비율 계산 · WCAG 기준 확인
  • 업로드 메타데이터 · width/height context
LLM 비전

구조화된 facet 분석

  • 입력: 스크린샷 이미지 + 추출 palette
  • 스키마 출력: typography · layout · spacing · componentStyle
  • mood keywords: 소스 조화도 계산 입력
  • strict JSON schema · 자유 형식 parsing 없음
  • confidence/evidence → 출처 배지

일관성 점수 로직: 규칙 기반 기준

일관성 점수 = clamp(100 - Σ dimension 감점)
접근성text/background < 4.5:1 · primary/background < 3:1
시각 일관성compact + body < 14px · 간격/밀도 불일치
의도 커버리지palette · typography · layout · spacing 누락
출처 커버리지선택 source 근거 누락
소스 조화도mood keyword Dice similarity · 낮은 confidence source
생성 준비도빈 프롬프트 · 빈 화면 계획
수정 계획
  • 대비 수정: 텍스트 색상을 WCAG AA 기준으로 조정
  • 타이포그래피 수정: compact layout body 최소 14px
  • 간격 수정: compact 4px · comfortable 8px
Recipe 순위화
  • 후보 조합 ≤ 200
  • 정렬: 일관성 점수 높은 순
  • tie-break: 섞인 source 수가 적은 순

Judge 프롬프트 가드레일

  • version: coherence-judge-v2
  • rating: strong · adequate · weak · fail
  • 규칙 finding = 사실 힌트, 숫자 anchor 아님
  • 출처 근거 임의 생성 금지

운영 모드

  • judgeMode: shadow frontend 평가 호출에 포함
  • primary mode disabled
  • judge 실패 → warning log · baseline result 유지
  • feedback 저장: coherence-feedback.json

API

Fastify가 업로드, facet 추출, intent 평가, v0 생성, audit을 /api/... 경로로 연결합니다.

테스트 코드

Orchestration

별도 agent framework 대신 route와 typed artifact로 분석, 계획, 검증, 생성, 감사를 순서대로 실행합니다.

일관성 점수

대비, 간격, 출처, 커버리지 mismatch를 감점해 생성 전 repair plan을 제안합니다.

Audit

생성된 코드가 IntentSpec과 facet provenance를 얼마나 반영했는지 생성 후 다시 점검합니다.

핵심 차별점

StylePrint는 레퍼런스를 facet 단위로 분해해 조합하고, coherence로 생성 전 충돌을 줄이며, audit으로 생성 후 결과의 의도와 출처까지 추적하는 UI 생성 시스템입니다.

마무리: 차별점과 발전 방향

차별점

  • Facet 단위 추출: 색상, 레이아웃, 간격, 컴포넌트 스타일 분리

  • JSON 저장소를 DB와 object storage 기반으로 전환

  • Planner, Validator, Generator를 더 명확한 multi-agent 구조로 분리
